Solution steps

Step 1: image analysis

I started by doing a basic analysis of the image.

$ file ./red.png
./red.png: PNG image data, 128 x 128, 8-bit/color RGBA, non-interlaced

It seemed to be a real PNG image. I extracted the strings inside it.

$ strings ./red.png
IHDR
tEXtPoem
Crimson heart, vibrant and bold,
Hearts flutter at your sight.
Evenings glow softly red,
Cherries burst with sweet life.
Kisses linger with your warmth.
Love deep as merlot.
Scarlet leaves falling softly,
Bold in every stroke.x
IDATx
IEND

It came out as a kind of love poem. I realized that, by looking at the first letter of each line, it hides a message that says: "CHECK LSB". So, the advice was to check the LSBs of the image pixels.

Every image is made up of a grid of pixels and each pixel contains information about the color to be displayed at that specific point. Most images use the RGB color model, which stands for Red, Green, Blue (or RGBA, where A stands for Alpha, indicating the level of transparency). This is an additive color model used to represent colors on a screen, that are obtained by combining different intensities of red, green, and blue.

Each component (also referred as color channel) of the tuple [R, G, B], has a value ranging from 0 to 255. Therefore, in computing terms, the information for each component is represented by one byte.

The LSB of each of these bytes can be modified and used to hide information. This method of Steganography (LSB Steganography) will alter the color of each pixel, but only slightly. In fact, it's hardly noticeable to the naked eye and the image would look the same as before.

Step 2: exploitation

For the reasons just mentioned, I developed a Python script to try extracting the LSBs. You can find a more general version of it here.

import numpy as np
from PIL import Image

image_path = "./red.png"
image  = Image.open(image_path, "r")
pixels = np.array(image.getdata())

num_channels   = 0
ls_bits        = ""
hidden_message = ""

if   image.mode == "RGB":  num_channels = 3
elif image.mode == "RGBA": num_channels = 4
else:
    print("Cannot process this image because it's not supported.")
    exit(0)

# num_pixels = len(pixels)
num_pixels = pixels.size // num_channels

# create a concatenation of all the LSBs (for each channel of each pixel)
for i in range(num_pixels):
    for j in range(num_channels):
        ls_bits += bin(pixels[i][j])[-1]

# from the previous concatenation, create a list of 8-bits chunks
ls_bits = [ls_bits[i:i+8] for i in range(0, len(ls_bits), 8)]

# from the previous list, create a concatenation of all the ASCII characters
# corresponding to the chunks
for i in range(len(ls_bits)):
    hidden_message += chr(int(ls_bits[i], 2))

print(hidden_message)

Personal notes

  • In LSB Steganography, information can be hidden using any possible subset of the available channels: e.g., [R, G], [R, B], [G], [R, G, B, A], [B, A]. Additionally, the message can be hidden starting from a certain offset and not necessarily from the first pixel and can end at a certain offset before the end and not necessarily at the last pixel.
